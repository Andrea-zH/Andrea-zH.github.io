# Hugo 网站本地预览 & 部署到 GitHub Pages 使用说明

## 一、操作前提

1. 已在 `D:\blog\andrea` 目录下完成 Hugo 网站（LoveIt 主题）的配置；
2. 本地已安装 Git、Hugo，且 `D:\blog\andrea` 目录已关联你的 GitHub 仓库（`andrea-zh.github.io`）；
3. 操作工具：Windows 命令提示符（CMD）/PowerShell。

## 二、分步操作说明

### 步骤 0：创建第一篇博客文章（新增）

1. 确保 CMD/PowerShell 已进入 Hugo 网站根目录：
    
    bash
    
    运行
    
    ```
    # 切换到 D 盘
    D:
    # 进入 Hugo 网站根目录
    cd D:\blog\andrea
    ```
    
2. 执行创建文章的命令：
    
    bash
    
    运行
    
    ```
    hugo new posts/hello-world.md
    ```
    
    - 执行成功后，会在 `D:\blog\andrea\content\posts` 目录下生成 `hello-world.md` 文件；
    - 文件默认包含基础的 Front Matter 模板（如下），`draft = true` 表示文章处于草稿状态：
        
        toml
        
        ```
        +++
        date = '2026-03-04T09:18:29+08:00'
        draft = true
        title = 'Hello World'
        +++
        ```
        
    
3. 编辑文章内容：
    
    - 可以用 Obsidian 打开 `D:\blog\andrea\content\posts\hello-world.md` 文件，在 `+++` 下方编写 Markdown 格式的正文；
    - 若想发布文章（非草稿），可将 `draft = true` 改为 `draft = false`。
    

### 步骤 1：本地预览 Hugo 网站（验证配置 / 内容效果）

1. 保持在 `D:\blog\andrea` 目录下（若已退出，重新执行目录切换命令）；
2. 执行本地预览命令：
    
    bash
    
    运行
    
    ```
    hugo server 
    ```
    
    - 参数 `-D`：显示草稿状态的文章（如果不需要看草稿，可省略；若省略，`draft = true` 的文章不会显示）；
    - 命令执行成功后，终端会提示服务启动成功，默认访问地址：`http://localhost:1313`。
    
3. 打开浏览器，输入 `http://localhost:1313`，即可实时查看网站效果（包括刚创建的 `hello-world.md` 文章，修改内容后刷新浏览器就能看到更新）。
4. 停止本地服务：在 CMD/PowerShell 中按 `Ctrl + C` 即可。

### 步骤 2：构建 Hugo 静态网页（生成部署文件）

1. 确保当前仍在 `D:\blog\andrea` 目录下（若已退出，重新执行步骤 1 的目录切换命令）；
2. 执行构建命令（无需额外参数，会自动生成 `public` 文件夹）：
    
    bash
    
    运行
    
    ```
    hugo
    ```
    
    - 执行完成后，`D:\blog\andrea` 目录下会生成 `public` 文件夹，里面是完整的静态网页文件（这是最终要部署的内容）；
    - 注意：若文章仍为 `draft = true`，构建时不会包含该文章，需先改为 `draft = false` 再执行 `hugo`。
    

### 步骤 3：部署到 GitHub Pages（推送到 gh-pages 分支）

1. 进入生成的 `public` 目录：
    
    bash
    
    运行
    
    ```
    cd D:\blog\andrea\public
    ```
    
2. 提交构建结果到 GitHub 仓库：
    
    bash
    
    运行
    
    ```
    # 将所有静态文件加入 Git 暂存区
    git add .
    # 提交修改（备注信息可自定义，比如 "新增Hello World文章 2026-03-04"）
    git commit -m "手动部署 Hugo 页面"
    # 强制推送到 gh-pages 分支（-f 确保覆盖旧版本，GitHub Pages 通常读取该分支）
    git push -f origin gh-pages
    ```

**3.17 乱改导致push为：git push origin HEAD:gh-pages --force**

1. 访问部署后的网站：推送成功后，等待 1-5 分钟（GitHub 同步时间），打开浏览器访问 `https://andrea-zh.github.io/` 即可查看部署效果。

## 三、常见注意事项

1. 若执行 `hugo new posts/hello-world.md` 提示「找不到目录」：检查 `content/posts` 目录是否存在，若不存在可手动创建，或确保 Hugo 版本为最新；
2. 若执行 `hugo` 命令后 `public` 文件夹为空：
    
    - 检查 `hugo.toml` 配置是否正确，确保 `theme` 字段指定了正确的主题名称（如 `theme = "LoveIt"`）；
    - 确认要发布的文章 `draft = false`；
    
3. 推送失败（Git 报错）：
    
    - 确认本地 Git 已配置 GitHub 账号（用户名 / 邮箱）；
    - 确认仓库地址正确，且有 `gh-pages` 分支（若无，可先执行 `git checkout -b gh-pages` 创建）；
    
4. 访问 `https://andrea-zh.github.io/` 显示 404：
    
    - 检查 GitHub 仓库设置：仓库→Settings→Pages→Build and deployment→Source 选择 `gh-pages` 分支，根目录（/），保存后重新推送；
    - 等待 GitHub Pages 同步完成（最长不超过 10 分钟）。
    

### 总结

1. 新建文章核心命令：`hugo new posts/hello-world.md`（根目录执行），生成的文件在 `content/posts` 下，可通过 Obsidian 编辑；
2. 本地预览核心命令：`hugo server -D`（`-D` 显示草稿文章），访问 `http://localhost:1313` 验证效果；
3. 构建部署关键：`hugo` 生成 `public` 文件夹，进入该目录后通过 `git add/commit/push` 推送到 `gh-pages` 分支，最终访问 `https://andrea-zh.github.io/` 查看结果。
