# Hugo 博客使用说明

## 分支说明

| 分支 | 用途 | 操作对象 |
|------|------|----------|
| **master** | 存储博客源码 | 你写文章、提交代码的地方 |
| **gh-pages** | 存储构建产物 | GitHub Actions 自动部署，**不要手动操作** |

**重要：永远只操作 master 分支，不要手动提交到 gh-pages 分支！**

---

## 快速上手

### 第一步：写文章

1. 进入文章目录：`D:\blog\andrea\content\posts`
2. 新建 `.md` 文件（或用命令创建：`hugo new posts/文章名.md`）
3. 文件开头添加 Front Matter：

```toml
+++
date = '2026-03-20T10:00:00+08:00'
draft = false
title = '文章标题'
+++
```

4. 在 `+++` 下方用 Markdown 写正文
5. 图片放在文章同级目录，引用方式：`![描述](图片名.jpg)`

### 第二步：本地预览（可选）

```bash
cd D:\blog\andrea
hugo server -D
# 浏览器访问 http://localhost:1313
# Ctrl+C 停止服务
```

### 第三步：发布到网站

```bash
# 1. 进入项目目录
cd D:\blog\andrea

# 2. 查看修改内容（确认无误）
git status

# 3. 添加所有修改
git add .

# 4. 提交修改（引号内写说明）
git commit -m "新增春菜文章"

# 5. 推送到 master 分支（重要！不是 gh-pages）
git push origin master
```

等待 1-2 分钟，访问 `https://Andrea-zH.github.io/` 即可查看。

## 工作流程

```
你写的 Markdown 文件 (master 分支)
        │
        │ git push origin master
        ▼
   GitHub 仓库
        │
        │ GitHub Actions 自动触发
        ▼
   hugo --minify 构建
        │
        │ 自动部署
        ▼
   gh-pages 分支
        │
        │ GitHub Pages 提供访问
        ▼
https://Andrea-zH.github.io/
```

## 常见场景

### 发布新文章
```bash
cd D:\blog\andrea
git add .
git commit -m "新增文章：XXX"
git push origin master
```

### 修改已有文章
```bash
cd D:\blog\andrea
git add .
git commit -m "修改文章：XXX"
git push origin master
```

### 删除文章
```bash
cd D:\blog\andrea
git rm content/posts/要删除的文章.md
git commit -m "删除文章：XXX"
git push origin master
```

### 添加图片
1. 将图片放入文章同级目录（Page Bundle 模式）
2. 在 Markdown 中引用：`![描述](图片名.jpg)`
3. 提交并推送

---

## 注意事项

1. **分支问题**：只推送到 `master` 分支，不要推送到 `gh-pages`
2. **草稿文章**：`draft = true` 的文章不会发布，需改为 `draft = false`
3. **图片路径**：Page Bundle 模式下直接引用文件名，不要加目录前缀
4. **部署失败**：查看 GitHub 仓库的 Actions 标签页，检查错误日志
5. **网站 404**：检查 Settings → Pages → Source 是否选择 `gh-pages` 分支

## 紧急情况

### 如果不小心推送到 gh-pages 分支

```bash
# 重新构建并部署到 gh-pages
hugo --minify
cd public
git add .
git commit -m "重新部署"
git push origin HEAD:gh-pages --force
```

### 如果网站图片不显示

1. 检查图片路径是否正确
2. 等待 30 秒让 GitHub Pages 更新
3. 如果还不行，手动部署：
```bash
hugo --minify
cd public
git add .
git commit -m "手动部署"
git push origin HEAD:gh-pages --force
```
